Guile Scheme Overview

20.04.2022 03:00

Guile Scheme is actively maintained scheme dialect by the GNU project and has receieved continued improvement over the years. Most recently, version 3 introduced JIT compilation to speed it’s performance considerably.

Like (some) other Lisps, allows you to interact with and modify your program while it’s running. It also has good C interop for both C FFI or for embedding in (and adding interactivity to) C programs.

If you’re interested in using GNU GUIX, the entire GUIX system as well as the Shepherd init system are written in Guile. It’s under very active development and is a lot of fun to hack on, so if tinkering with a Lisp-based OS sounds appealing to you, I highly recommend you check it out.

Being a GNU project, it’s hackable by design and very well documented. It has an excellent manual version and the source code is literred with useful comments, readmes, and other breadcrumbs that I’ve really come to appreciate.

There are plenty of libraries worth checking out, such as Haunt Static Site Generator, Chickadee Game Development Toolkit, or the Artanis Web Framework, all of which are recent projects and actively maintained.

If you’re looking to pick up a general-purpose scheme for learning purposes, it’s sufficient. Racket is better for that, honestly, because of the great tooling, debugging tools, cross-platform availability, and how everything “just works”. But now that I am familiar with both, I stick to Guile, because the advantages of Racket don’t matter as much as you become more knowledgeable (in my opinion at least).

How is the tooling?

If you’re used to the tooling of lisps like Racket, Clojure, or CL, Guile’s tooling is not as polished. Within Emacs, goto-definition, autocomplete, and in-editor evaluation are available with Geiser. Breakpoints and step debugging are available, but not within Geiser. The documentation on interactive debugging is available here.

Package management is accomplished with the module system. Deploying libraries means adding files to installation-designated directories- compiled files are put in the SITE_CCACHE_DIR and source files to the GUILE_SITE_DIR. Most libraries that I’ve seen use autotools (the tools behind ./configure, make, make install) to figure out where these are. If you’re on a Mac and unfamiliar with Guile, you’ll want to lean on the homebrew links below and read them carefully to make sure stuff works as expected.

Installation Tips

Installation is very straightforward on both Linux and MacOS. Below are a few installation tips for installing Guile 3 (which is designed to co-exist with older versions, so don’t worry about conflicts).

Arch Linux

Install Guile 3 from the AUR package.

Ubuntu

Guile 3 is available as a Ubuntu package

MacOS

Guile is available on homebrew. There’s also a very handy homebrew tap for installing popular Guile libraries and applications.

Post Installation

On both my Linux and Mac, I added the “guile site dir” to guile’s load path by adding a ~/.guile file, which on my mac contains:

(add-to-load-path "/opt/homebrew/share/guile/site/3.0")
(add-to-load-path "/usr/local/share/guile/site/3.0")

These are where homebrew libraries and libraries that I manually built are found, respectively.

Also, I needed to tell Geiser to use this file for it to work. Here’s my use-package declaration for guile-geiser:

(use-package geiser-guile
  :ensure t
    :config
      (setq geiser-guile-load-init-file-p t))
